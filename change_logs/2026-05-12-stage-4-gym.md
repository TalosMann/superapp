# Stage 4 — Gym module (workouts + sports + progress)

The biggest module yet. Workout logging with sets/reps/weight, sport logging in three different modes (duration, distance, sets), per-exercise and per-activity progress charts, library that grows as you log, automatic goal-counter incrementing.

## What's in this stage

### Three views in the Gym tab

- **Sessions** — chronological log of all workouts + sport sessions, grouped by date, newest first. Tap any to edit or delete. Empty state on first load.
- **Exercises** — your lifting exercise library (Bench, Squat, Dips, etc.). Search. Tap an exercise → progress page for it. Library grows as you log; you don't have to pre-define anything.
- **Activities** — your sport library (Dodgeball, Zigzag run, Plyometrics, etc.). Same UX. Tap → progress page.

### Two loggers (FABs)

Red dumbbell button: log a workout. Yellow ⚡ button: log a sport session. Both only appear on the Sessions tab.

### Workout logging

1. Pick a workout type — automatically populated from your timetable's Gym-category events (Push, Pull, Legs, etc.). Plus a "Custom" option.
2. If a previous session of the same type exists, you'll be prompted: **"Copy last Push session?"** — prefills exercises and weights from last time, so logging takes 30 seconds instead of 3 minutes.
3. Add exercises from the library (or "Quick add" inline if it's new).
4. For each exercise, log sets: weight × reps (or just reps for bodyweight). Each new set defaults to "same as previous" so you usually just need to confirm.
5. Save.

### Sport logging — three modes

Each activity has a mode set when you create it. The session form adapts:

- **Duration mode** (Dodgeball, Juggling, Ball Control): just enter minutes
- **Distance + duration** (Swimming, Zigzag run): distance + unit + minutes. Pace auto-derived.
- **Sets mode** (Plyometrics, Max Speed sprints, Curling throw): list of sets, each with optional reps/seconds/distance. Like workout sets but for drills.

### Progress views

Tap any exercise or activity from the library to see:

- **Top stat cards** — estimated 1RM (or top reps for bodyweight), heaviest set, sessions count
- **Line/bar chart** — progress over time, kind chosen based on data type
- **Session history** — every past session for this thing, newest first, tappable to open the session

### Goal auto-increment

When you save a new workout/sport session, the app walks your active counter goals. If a counter goal is linked to a timetable event whose `groupId` matches the workout type (for workouts) or whose title contains the activity name (for sports), the counter gets +1.

This means: a Weekly counter goal "Do 4 gym sessions" linked to your Push event will auto-increment every time you log Push. You can still increment manually too.

Conservative behavior:
- Counter goals only (not checkboxes — those stay manual)
- Only on new sessions, never on edits
- Maxes out at the target (won't go past)
- Skipped if already complete

### Home dashboard — Training section

Below Goals, a new "Training" card shows last-7-days summary: number of workouts, number of sport sessions, and a green "✓ Logged X today" if you've been today.

## Starter libraries

On first launch, the libraries are seeded:

**Exercises:** Bench press, OHP, Dips, Triceps pushdown, Pull-ups, Barbell row, Lat pulldown, Barbell curl, Squat, Deadlift, RDL, Single-leg squat.

**Activities:** Dodgeball, Ball juggles, Swimming, Zigzag run, Plyometrics, Max speed sprints, Shot power, Ball control, Curling throw — each with its mode pre-set based on how you described them.

You can edit/delete/add any of them. Once you log a session you'll see them in the library views with session counts and progress indicators.

## What I'd flag

**The starter seeding only runs once.** First time the Gym tab opens with no saved library, it writes the starters. After that, even if you delete everything, they don't come back. This is intentional — you don't want them resurrecting after you've curated your own list.

**Sport mode is fixed per activity.** You can't change "Zigzag run" from distance to sets later — you'd create a separate "Zigzag drill" activity for the sets-based version. As discussed, this is by design: the progress trail makes no sense if the metric changes.

**Library editing** (rename, change mode, delete from library) isn't built into this stage yet. For now, the libraries are append-only via the loggers. A "manage library" view would be a Stage 4b polish if you find yourself needing it. Honestly may not — items live forever just fine if you don't use them.

**Auto-increment is best-effort matching.** For a sport goal to auto-increment, the linked timetable event's title must contain the activity name OR vice versa. So "Dodgeball training" linked event with "Dodgeball" activity works. "DBall practice" linked event with "Dodgeball" activity does NOT. If this becomes annoying, we add an explicit "this goal counts a session of <activity>" field to goals.

**No "today's sessions" surfacing on the Timetable view yet.** If you log a Push session today, it doesn't appear inside the Push event card on the timetable. Possible Stage 4b polish.

## Try it

1. Open Gym → empty Sessions view
2. Tap the red dumbbell FAB
3. Pick workout type — if your timetable has Push (Armour) at category=gym, you'll see it as a chip. If not, tap "+ Custom" and type something.
4. Add Bench press from library, log 3 sets of `80 × 8`. Add Dips, 3 sets of `10` reps (no weight needed — it's bodyweight).
5. Save. Goes into Sessions list.
6. Switch to **Exercises** tab → tap Bench press → see your first data point on the chart. Log another session next week and you'll see the line move.
7. **Sport test:** tap the yellow ⚡, pick Dodgeball, enter 45 minutes, save. Switch to Activities → tap Dodgeball → see the duration chart.
8. **Goal integration test:** make a counter goal "Do 4 gym sessions this week", link to your Push event, target 4. Now log a Push workout — go back to Goals tab, you'll see the counter at 1/4.
